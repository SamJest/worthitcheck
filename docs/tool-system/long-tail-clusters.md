# WorthItCheck Long-Tail Clusters

This document keeps long-tail SEO work as structured as tool work. The goal is to turn each core tool into a small cluster of narrowly targeted pages that match real search intent and feed users back into the main calculator or tool.

## Page types

### 1. Hub page
- Targets a broad cluster phrase.
- Lists the strongest scenario pages in that cluster.
- Links users into the main calculator or tool.

### 2. Scenario page
- Targets a single long-tail query such as `repair or replace dishwasher`.
- Gives a fast practical answer, common decision factors, realistic examples, and FAQs.
- Links back to the main calculator plus adjacent scenario pages.

## Cluster workflow

1. Pick one existing tool with broad expansion potential.
2. Create one hub page for that tool's search cluster.
3. Create 4 to 8 scenario pages around the strongest item-specific queries.
4. Link the homepage or main tool page to the hub and top scenario pages.
5. Add every cluster URL to the sitemap.
6. Reuse one consistent page format so expansion stays fast.

## Long-tail page checklist

- Exact-match or close-match phrase in the title, H1, and URL
- Unique intro copy with item-specific decision signals
- At least 4 realistic example scenarios
- FAQ block with item-specific questions
- Clear call to use the main calculator
- Internal links to the hub and adjacent scenario pages
- Canonical, Open Graph, Twitter, and JSON-LD present

## Rule of thumb

If a new keyword can be answered with mostly the same tool but more specific context, make it a long-tail scenario page before building a whole new standalone tool.
