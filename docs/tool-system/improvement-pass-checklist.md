# Improvement Pass Checklist

Complete one fixed pass on existing tools during every new-tool cycle.

## Required pass types
- 1 UX improvement
- 1 logic / scoring improvement
- 1 SEO / content improvement
- 1 analytics or monetization improvement if applicable

## Review prompts
- Are users likely to understand the inputs quickly?
- Are borderline results explained clearly enough?
- Are example cases believable?
- Is there an obvious next click after the result?
- Is anything inconsistent with the rest of the site?

## Execution rules
- Prefer small fixes that can ship every cycle.
- If an issue appears in one tool, check the other tools for the same pattern.
- Log the improvement in `roadmap.md` so recurring issues become visible.
