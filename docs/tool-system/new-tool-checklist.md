# New Tool Checklist

Use this checklist before shipping any new tool.

## Build and content
- Tool brief is complete before code changes start.
- New page uses the shared tool scaffold unless there is a documented reason not to.
- Title, meta description, hero copy, FAQs, and example scenarios are written before final QA.
- Canonical, Open Graph, Twitter, and structured data are present before launch.
- Tool is added to the tools index, footer links, and any relevant related-tools sections.

## Logic and UX
- Empty and invalid input states are handled clearly.
- Obvious verdict, opposite-end verdict, and borderline verdict cases were manually tested.
- Confidence language matches the rest of the site.
- Thinking state, loading label, and result reveal behavior feel consistent with live tools.
- The page gives users a clear next click to another relevant tool.

## QA
- At least 10 realistic example scenarios were checked against expected outcomes.
- Mobile layout was reviewed.
- Result card, examples toggle, and reasoning details were checked.
- Shared analytics initialization is present.
- Links and relative paths work from the new folder location.

## Release
- New URL is added to `sitemap.xml`.
- Candidate / next-up / improvement backlog entries are updated.
- Scorecard template is filled in after the page is reviewed.
- A short retrospective note is added to the roadmap.
